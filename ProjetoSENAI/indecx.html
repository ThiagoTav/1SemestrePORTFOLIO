SCRUM

No inicio do curso foi nos apresentado a metodologia SCRUM, Scrum é uma metodologia ágil de gerenciamento de projetos que visa melhorar a eficiência e a colaboração em equipes de desenvolvimento de software.é baseado em princípios de transparência, inspeção e adaptação. Ele divide o projeto em ciclos de trabalho chamados de "sprints", que são períodos fixos e curtos (geralmente de duas a quatro semanas) durante os quais a equipe se concentra em entregar um conjunto específico de funcionalidades.

Os principais papéis no Scrum são:

Product Owner (Proprietário do Produto): É responsável por definir as necessidades dos clientes e priorizar os itens do backlog do produto.
Scrum Master: É responsável por facilitar o uso adequado do Scrum na equipe e garantir que os princípios e práticas sejam seguidos.
Equipe de Desenvolvimento: É responsável por desenvolver o produto e entregar as funcionalidades planejadas para cada sprint.

Durante cada sprint, a equipe realiza reuniões diárias de sincronização chamadas de "daily scrum" para compartilhar o progresso, identificar obstáculos e planejar as atividades do dia. Ao final de cada sprint, ocorre uma revisão do sprint, onde as funcionalidades concluídas são demonstradas e feedback é recebido. Em seguida, há uma retrospectiva, onde a equipe analisa o sprint e identifica maneiras de melhorar o processo.

O Scrum enfatiza a colaboração, a adaptação às mudanças e a entrega contínua de valor aos clientes. Ele proporciona maior flexibilidade, transparência e eficiência na gestão de projetos, permitindo que a equipe se adapte às necessidades do cliente e do mercado ao longo do desenvolvimento do produto. E seus pilares são:

  • Transparência
  • Inspeção
  • adaptação

    HTML (HyperText Markup Language) é a linguagem de marcação padrão usada para criar páginas da web. É a estrutura básica e fundamental de um site e define a estrutura e o conteúdo de uma página da web.

    O HTML consiste em elementos e tags que são usados para marcar diferentes partes do conteúdo de uma página, como cabeçalhos, parágrafos, listas, imagens, links e muito mais. Essas tags são interpretadas pelos navegadores da web para renderizar e exibir o conteúdo corretamente.

    Os elementos HTML são organizados em uma estrutura hierárquica, com elementos pai e elementos filhos. Isso permite a criação de uma estrutura lógica e semântica para o conteúdo da página.

    Além disso, o HTML permite a incorporação de outros recursos, como folhas de estilo CSS para estilizar a página, scripts JavaScript para adicionar interatividade e elementos multimídia, como áudio e vídeo.

    Em resumo, o HTML é a base para a criação de páginas da web. Ele fornece a estrutura e o conteúdo da página, permitindo a formatação, a organização e a exibição correta do conteúdo nos navegadores da web.

    HTML Semântico

    é uma abordagem de marcação que consiste em usar elementos HTML de forma significativa e descritiva para melhorar a estrutura, acessibilidade e compreensão do conteúdo da página pelos navegadores, mecanismos de busca e desenvolvedores. Em vez de usar elementos genéricos como "div" ou "span" para agrupar e estilizar o conteúdo, o HTML semântico utiliza elementos específicos que refletem o propósito e a estrutura do conteúdo em si. Isso torna o código HTML mais expressivo e semântico, fornecendo um significado claro sobre a finalidade de cada elemento.

    Ao utilizar HTML semântico, você está comunicando de forma mais precisa o significado do conteúdo para os navegadores e mecanismos de busca, o que pode ter benefícios para a otimização de mecanismos de busca (SEO). Além disso, o HTML semântico também pode melhorar a acessibilidade, permitindo que leitores de tela e outros dispositivos entendam melhor a estrutura e o fluxo do conteúdo.

    'header': Representa o cabeçalho da página ou de uma seção.

    'nav': Representa uma seção de navegação.

    'main': Representa o conteúdo principal da página.

    'article': Representa um conteúdo independente, como um post de blog.

    'section': Representa uma seção genérica do conteúdo.

    'aside': Representa um conteúdo relacionado, geralmente colocado ao lado do conteúdo principal.

    'footer': Representa o rodapé da página ou de uma seção.

    'figure' e 'figcaption': São usados para exibir uma figura (como uma imagem) e uma legenda associada.
