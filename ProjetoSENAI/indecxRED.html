Redes de Computadores

Tipos de Redes

  • Rede de Área Pessoal (PAN - Personal Area Network):
    • É uma rede que conecta dispositivos pessoais em uma área muito próxima, geralmente dentro de um alcance de alguns metros. Exemplos comuns de PANs incluem a conexão entre um smartphone e fones de ouvido Bluetooth ou a conexão entre um computador e um teclado sem fio.

  • Rede de Área Campus (CAN - Campus Area Network):
    • É uma rede que abrange um campus universitário ou corporativo. Ela interconecta diferentes edifícios dentro da área do campus, permitindo o compartilhamento de recursos e a comunicação entre eles. Uma CAN é geralmente maior do que uma LAN, mas menor do que uma WAN.

  • Rede de Área Local (LAN - Local Area Network):
    • É uma rede que cobre uma área geográfica relativamente pequena, como um escritório, uma casa ou um prédio. É usada para conectar dispositivos próximos e permitir o compartilhamento de recursos, como impressoras, arquivos e acesso à Internet. As LANs são comumente usadas em ambientes domésticos e empresariais.

  • Rede de Área Metropolitana (MAN - Metropolitan Area Network):
    • É uma rede que abrange uma área geográfica maior, como uma cidade ou uma região metropolitana. Ela interconecta várias LANs e outros dispositivos em uma área mais ampla, fornecendo comunicação entre eles. Uma MAN geralmente é usada para interligar diferentes escritórios ou campi em uma cidade.

  • Rede de Área Ampla (WAN - Wide Area Network):
    • É uma rede que abrange uma grande área geográfica, como um país, continente ou até mesmo globalmente. A WAN conecta diferentes redes, como LANs e MANs, por meio de tecnologias como linhas dedicadas, satélites e conexões de Internet. A Internet é o exemplo mais conhecido de uma WAN.

    Topologia Física/Lógica

  • Topologia Física:
    • A topologia física de uma rede se refere à disposição física e ao layout dos dispositivos de rede, cabos e outros componentes. Ela descreve como os dispositivos estão fisicamente conectados uns aos outros e como os meios de transmissão, como cabos ou redes sem fio, são usados para estabelecer as conexões. Alguns exemplos comuns de topologias físicas são a topologia de barramento, estrela, anel e malha.

  • Topologia Lógica:
    • A topologia lógica de uma rede se refere ao caminho lógico que os dados seguem através da rede, independentemente da sua disposição física. Ela descreve como os dispositivos se comunicam uns com os outros através do uso de protocolos de rede e endereçamento lógico. Alguns exemplos comuns de topologias lógicas são a topologia em estrela, em árvore, em malha e ponto a ponto.

    IP

    IP (Internet Protocol) é um protocolo de comunicação utilizado na Internet para identificar e rotear pacotes de dados entre dispositivos em uma rede. O IP é responsável pela atribuição de endereços únicos a cada dispositivo conectado à rede e pela definição de como esses dispositivos se comunicam entre si.

    Cada dispositivo conectado à Internet, como computadores, smartphones, servidores, entre outros, possui um endereço IP exclusivo, que é composto por uma sequência de números separados por pontos. Por exemplo, um endereço IP pode ser representado como 192.168.0.1.

    IPV4/IP

    IPv4 (Internet Protocol version 4) é a quarta versão do Protocolo de Internet, que é amplamente utilizado para identificar e rotear dispositivos em rede pela internet. O IPv4 utiliza um formato de endereço de 32 bits, representado por quatro conjuntos de números separados por pontos (por exemplo, 192.168.0.1). Cada conjunto de números, chamado de octeto, pode ter um valor variando de 0 a 255, permitindo um total de aproximadamente 4,3 bilhões de endereços únicos.

    O sistema de endereçamento IPv4 possui limitações devido ao número limitado de endereços disponíveis. Com o contínuo crescimento da internet, a demanda por endereços IP tem excedido a oferta disponível. Esse problema levou ao desenvolvimento e adoção do IPv6 (Internet Protocol version 6), que oferece um espaço de endereçamento significativamente maior para acomodar o crescente número de dispositivos conectados à internet.

    O IPv6 utiliza um formato de endereço de 128 bits e oferece um número praticamente ilimitado de endereços únicos, o que é essencial para a expansão contínua da internet. No entanto, o IPv4 ainda é amplamente utilizado e continuará em uso no futuro previsível. Muitas redes e dispositivos suportam tanto o IPv4 quanto o IPv6, permitindo um período de transição durante o qual ambos os protocolos coexistem.

    MAC

    MAC (Media Access Control) é um endereço físico único atribuído a cada interface de rede de um dispositivo. É um identificador exclusivo gravado na placa de rede ou adaptador de rede durante a fabricação.

    Diferentemente do endereço IP, que pode ser alterado ou atribuído dinamicamente, o endereço MAC é fixo e não pode ser modificado. Ele é composto por 48 bits (6 bytes) e é geralmente representado por uma sequência de números e letras separados por dois-pontos ou hífens. Por exemplo, um endereço MAC pode ser 00:1A:2B:3C:4D:5E.

    O endereço MAC é usado principalmente na camada de enlace de dados em redes locais (LANs), como Ethernet. Ele é utilizado para identificar exclusivamente uma interface de rede em um segmento de rede. Os dispositivos de rede, como switches e roteadores, utilizam o endereço MAC para encaminhar os pacotes de dados corretamente dentro da rede local.

    É importante ressaltar que o endereço MAC opera em um escopo limitado à rede local, enquanto o endereço IP é usado para comunicação em uma rede mais ampla, como a Internet. O endereço MAC não é roteável pela Internet, pois é substituído pelo endereço IP nas comunicações fora da rede local.

    Modelo OSI e TCP/IP

    O Modelo OSI (Open Systems Interconnection) é um modelo de referência para protocolos de comunicação em redes de computadores. Ele foi desenvolvido pela ISO (International Organization for Standardization) e é amplamente utilizado como uma base para entender e implementar redes de computadores. O modelo OSI divide o processo de comunicação em sete camadas distintas, cada uma com sua função específica. Aqui está uma descrição das sete camadas do modelo OSI:

  • Camada Física:
    • É a camada mais baixa do modelo OSI e lida com a transmissão de bits brutos por meio de meios de comunicação físicos, como cabos de cobre, fibras ópticas ou ondas de rádio. Ela define as características elétricas, mecânicas e funcionais das interfaces físicas entre os dispositivos de rede.

  • Camada de Enlace de Dados:
    • Esta camada fornece a transferência confiável de dados entre dispositivos adjacentes em uma rede. Ela detecta e corrige erros de transmissão, organiza os dados em quadros (frames) e controla o acesso ao meio físico.

  • Camada de Rede:
    • É responsável pela transmissão de pacotes de dados pela rede. Ela define protocolos para roteamento de pacotes, encaminhamento e controle de congestionamento. O endereçamento lógico (como o endereço IP) é realizado nesta camada.

  • Camada de Transporte:
    • Esta camada garante a entrega confiável e ordenada dos dados transmitidos entre hosts (dispositivos finais) na rede. Ela divide os dados em segmentos, estabelece conexões lógicas, controla a entrega e a sequência dos segmentos, e lida com o controle de fluxo e a correção de erros.

  • Camada de Sessão:
    • A camada de sessão estabelece, gerencia e encerra as sessões de comunicação entre os aplicativos em diferentes hosts. Ela sincroniza a comunicação, mantém o controle de diálogos e permite a recuperação em caso de falhas.

  • Camada de Apresentação:
    • Esta camada é responsável pela representação e formatação dos dados transmitidos. Ela lida com a tradução e a codificação de diferentes formatos de dados, compressão, criptografia e outras funções de manipulação de dados.

  • Camada de Aplicação:
    • A camada de aplicação é a camada mais alta do modelo OSI e fornece serviços de rede aos aplicativos do usuário final. Ela inclui protocolos para serviços como correio eletrônico (SMTP), transferência de arquivos (FTP), navegação na web (HTTP) e muitos outros.

    O modelo TCP/IP é um modelo de referência para protocolos de comunicação em redes de computadores. Ele é amplamente utilizado como base para a implementação da Internet e é composto por quatro camadas principais.
  • Tipos de Protocolos: HTTP, HTTPS, FTP, ICMP, FTPS.
